==**Примечание**:== по ходу объяснения работы логических операторов в go я использую значения 0 или 1, воспринимайте их как 0 =  false, а 1 = true 
Иногда нам нужно,чтобы при тех или иных значениях переменной,программа вела себя по-разному,с этим помогают операторы ветвления
Пример синтаксиса:
```go
package main

import "fmt"

func main() {
	score := 5
	if score > 10 {
		fmt.Println("Ты красавчик")
	} else {
	fmt.Println("Тебе еще надо учиться")
	}
}
```

if отвечает за выполнение программы при том или ином условии,else при ином случае

==**Примечание**:== в коде выше при score = 10 будет работать блок кода после else, так как 10 не больше 10,чтобы у нас работал блок кода после условия,нам надо использовать оператор ">=" 

## Что делать при нескольких условиях ?
Допустим,что у нас есть задача,при которой у нас несколько условий,например у нас есть некая игра,где при количестве очков >10 мы должны вывести "Ты красавчик",но если количество очков >15,то выводится "Ты супер красавчик"
Один из видов реализации ниже:
```go
package mainПримечание:

import "fmt"

func main() {
	score := 5
	if score > 10 {
		if score > 15 {
			fmt.Println("Ты супер красавчик")
		} else {
			fmt.Println("Ты красавчик")
		}
	} else {
		fmt.Println("Иди гуляй")
	}
}
```  
Но у нас тут проблема в том,что много вложенности,от чего читать код сложнее,как реализовать легче описано в пункте ниже
## else if
поэтому можно написать это все легче:

```go
score := 13
if score > 15 {
	fmt.Println("Ты мегакрасавчик")
} else if score > 10 {
	fmt.Println("Ты красавчик")
} else {
	fmt.Println("учись")
}
```
Что мы тут видим? во первых,код стал куда меньше, во вторых, вложенности меньше,она теперь одиночная,в третьих, алгоритм другой,если в первом варианте мы шли от неправильного к правильному,то сейчас от правильного к неправильному, ну в в четвертых появился какой-то *else if*, из себя он представляет по сути конструкцию,при которой у нас у ином случае от главного условия есть еще одно, и чтобы не увеличивать вложенность и сделать код более читаемым,рационально использовать его 

Для более наглядного примера,приведу задачу,где используется несколько else if: У нас есть игра,при которой если количество очков =12,то выводится "Дюжина", если =21, то выводится "Очко", если 50,то "Полтинник", если ничего из этого,то "Ты не попал никуда"
Реализация:
```go
score := 50
if score == 12 {
	fmt.Println("Дюжина")
} else if score == 21 {
	fmt.Println("Очко")
} else if score == 50 {
	fmt.Println("Полтинник")
} else {
	fmt.Println("Ты не попал никуда")
}
```

## Логические конструкции

По сути условные конструкции(>,  <, " == ", >=, <=, !=, ||, &&) показывают нам булево значение (true,false), как это работает?  Возьмем в пример задачу,по которой у нас при подписке на канал(true) выводится текст "ты подписан на мой ютуб канал, спасибооо!!!!!", в ином случае (false) выводится  "подпишись пж"
Реализовать это можно так:
```go
subscribed := true
if subscribed {
	fmt.Println("ты подписан на мой ютуб канал, спасибооо!!!!!")
} else {
	fmt.Println("подпишись пж")
}
```

То есть,у нас все сходится к логическому сравнению,короче говоря блок *if*  срабатывает при значении true, в случае если нет значения true, то срабатывает блок *else*

Для наглядности ниже пример:

```go
number := 5
ravno5 := number == 5
bolshe12 := number > 12
fmt.Println(ravno5) // true
fmt.Println(bolshe12) // false
```

## Сложные условные выражения
Чтобы объяснить возьму задачу: в нашей игре при наборе меньше 6 очков ИЛИ больше 16 выводится на экран "шестерка", тоесть, от 6 до 16 у нас не должно ничего выводится, так как нет объединения,то используется 
Реализация:

```go
score := 5
if score < 6 || score > 16 {
	fmt.Println("ты шестерка")
}
```


Выше был пример с оператором ИЛИ, ниже пример с И, по условию если мы попали с промежуток больше восьми и меньше 15,то выводится "ты попал в яблочко"
```go
if 8 < score && score < 15 {
	fmt.Println("Ты попал в яблочко")
}
```

## Алгебра логики 
Вообще,чтобы в это вникнуть,нужно знать алгебру логики,но есть вкратце,то ниже объяснение:
Пусть у нас есть логические операнда a=1 и b=0:
если у нас or(||), a || b = 1, то есть нужно,чтобы один из операндов был true,
если and(&&), a && b = 0, тоесть для того,чтобы получилось true, нужно чтобы 2 операнда было true, в ином случае(00,01,10) будет false
![[logic.png]]

Пример в go:
```go
// с ||

computerClub := false
icecream := true
if computerClub || icecream {
	fmt.Println("Я иду гулять") // выведется это
} else {
	fmt.Println("Я дома")
}

  

// с &&

sunny := false
weekend := true
if sunny && weekend {
	fmt.Println("Я иду гулять")
} else {
	fmt.Println("Я дома") // выведется это
}
```

## Операторы >= и <=

Отличаются от операторов больше-меньше только тем,что включают в сравнение цифру из условия, допустим у нас есть задача,по которой при которой ОТ 10 и больше очков выводится "Ты красава",это можно реализовать через score>9,но можно и так:

```go
score := 0
if score >= 10 { 
	fmt.Println("Ты красава")
}
```

Теперь мы "красавы" только если у нас очков от 10 и меньше, тогда:
```go
score := 0
if score <= 10 { 
	fmt.Println("Ты красава")
}
``` 

## Оператор "не равно"(!=)
Допустим у нас задача,по которой игрок если набирает ровно 7 очков,то он проигрывает,но если НЕ 7 очков,то выигрывает, то есть "трушное" значение - это счет не равно 7,тогда:
```go
if score != 7 {
	fmt.Println("Ты победил")
} else {
	fmt.Println("Ты проиграл")
}
``` 

## Логическое отрицание(!)
Проще говоря инвертирует логическое значение в обратное.
Пусть a = 0, тогда !a = 1; И наоборот,если a = 1, то !a = 0

В качестве примера задачи: если человек НЕ подписан,то пригласить его подписаться

```go
subscribed := false
if !subscribed {
	fmt.Println("Подпишись") // выведется это 
} else {
	fmt.Println("уже подписан")
}
```
