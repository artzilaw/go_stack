Структура - это логическое объединение нескольких разных переменных над которыми можно производить какие-то действия
![[struct.png]]

Допустим нам надо как-то создать сущность пользователя, имя у него типа строка, рейтинг типа дробное число и тд, вот тут то и понадобится структура
![[user_struct.png]]
Также, мы можем выполнять какие-то действия над полями(переменными) этой структуры

Как это выглядит в go:
```go
type User struct {
	Name string
	Age int
	PhoneNumber string
	IsClose bool
	Rating float64
}
```

Но это лишь описание User, создается он так:
```go
package main

import "fmt"

type User struct {
	Name string
	Age int
	PhoneNumber string
	IsClose bool
	Rating float64
}

func main() {
	user := User{}
	fmt.Println("User:", user)
}
```

Но в коде выше  полям будут заданы значения по умолчанию(User: { 0  false 0}) , так как мы не задали значения полей для User, если задать, то все будет выводиться корректно:
```go
user := User{
	Name: "artzilaw",
	Age: 20,
	PhoneNumber: "89882281488",
	IsClose: true,
	Rating: 8.0,
}

при выводе структуры будет:
User: {artzilaw 20 89882281488 true 5}
```

 Если надо обратиться к какому-то из полей, то пишут переменную, которой задана структура и через точку название поля:
 ```go
 fmt.Println("Имя:", user.Name) //вывод имени 
 ```
Также поля подвержены изменениям:
```go
fmt.Println("Имя до:", user.Name) // artzilaw
user.Name = "Мага"
fmt.Println("Имя до:", user.Name) // Мага
```

## Методы

Но по-хорошему так,как выше описывать структуру нельзя,так как поля открыты и их могут изменить не так,как задумано, для этого существуют методы,но пока представим,что у нас есть программа, где можно приветствовать и прощаться с пользователем, это можно реализовать так:
```go
package main

import "fmt"

type User struct {
	Name string
	Rating float64
}

func Greeting(u User) {
	fmt.Println("Всем привет!")
	fmt.Println("Меня зовут,", u.Name)
	fmt.Println("Мой рейтинг:", u.Rating)
	fmt.Println("")
}

func Goodbye(u User) {
	fmt.Println("Всем пока!")
	fmt.Println("Меня звали,", u.Name)
	fmt.Println("Мой рейтинг был:", u.Rating)
	fmt.Println("")
}

func main() {
	user := User{
		Name: "Вася",
		Rating: 5.0,
	}
	Greeting(user)
}
```

То есть, мы через через функцию передали **копию** переменной user и с этой переменной уже работали функции Greeting() и Goodbye(), но если мы используем много функций и много файлов,то можно запутаться,вот поэтому и нужны методы

**==Примечание:==** тут опять же функции передается только копия, так что на оригинальную переменную результат ее работы никак не повлияет 

Тоесть,программу выше можно реализовать так:
```go
package main

import "fmt"

type User struct {
	Name string
	Rating float64

}

  

func (u User) Greeting() {
	fmt.Println("Всем привет!")
	fmt.Println("Меня зовут,", u.Name)
	fmt.Println("Мой рейтинг:", u.Rating)
	fmt.Println("")
}

  

func (u User) Goodbye() {
	fmt.Println("Всем пока!")
	fmt.Println("Меня звали,", u.Name)
	fmt.Println("Мой рейтинг был:", u.Rating)
	fmt.Println("")
}

  

func main() {
	user := User{
		Name: "Вася",
		Rating: 5.0,
	}
	user.Greeting()
	user.Goodbye()
}
```
С первого вида ничего не поменялось, но у нас теперь Greeting() и Goodbuy() методы структуры User{} и их можно вызывать непосредственно напрямую из переменной,которая привязана к структуре(	user.Greeting(),user.Goodbye())
По сути логически, что обычный метод задания функции,что метод задания метода(о да,тавтология) это одно и то же, но второе нам помогает понять,что конкретно относится к нужной структуре

## Изменение полей структуры через методы 

Как рассматривалось в [[06_Указатели | конспекте про указатели]], мы с помощью них можем менять через функцию значение нужной переменной, со структурами работает также, более того, ничего разыменовывать не надо, Go это делает автоматически:
```go
func RaitingUp(u *User, raiting float64) {
	if u.Rating+raiting <= 10 {
		u.Rating += raiting
		fmt.Println("Я добавил рейтинг")
	} else {
		fmt.Println("Я не прошел валидацию")
	}
}

func main() {
	user := User{
		Name: "Вася",
		Rating: 4.0,
	}
	fmt.Println("user до:", user)
	fmt.Println("")
	
	RaitingUp(&user, 5.0)
	
	fmt.Println("user после:", user)
	fmt.Println("")
}
```

 Но все же, это лучше делать через методы, механизм впринципе похож:
 ```go
 func (u *User) RaitingUp(raiting float64) {
	if u.Rating+raiting <= 10 {
		u.Rating += raiting
		fmt.Println("Я добавил рейтинг")
	} else {
		fmt.Println("Я не прошел валидацию")
	}
}

func main() {
	user := User{
		Name: "Вася",
		Rating: 4.0,
	}
	fmt.Println("user до:", user)
	fmt.Println("")
	
	user.RaitingUp(5.0)
	
	fmt.Println("user после:", user)
	fmt.Println("")
}
 ```

Отличий короче немного,но лучше вторым методом,чтобы это было "методом"

Кстати, эта конструкция( (u User) и (u * User)) называется **ресивером** , он может быть как по значению, так и по указателю(где что объяснять наверное не надо).
Если не надо менять исходную структуру, то надо использовать ресивер по значению, если надо, то по указателю

Теперь вернемся к нашему User, ниже представлена его реализация, согласно фоткам выше(+ смена имени):

```go
package main

import "fmt"

type User struct {
	// Имя пользователя
	// Правило: имя должно быть не пустое
	Name string
	// Возраст пользователя
	// Правило: Возраст должен быть больше 0 и меньше 150
	Age int
	// Номер телефона
	// Правило: номер телефона не должен быть пустым
	PhoneNumber string
	// Закрыт ли профиль пользователя?
	IsClose bool
	// Рейтинг пользователя
	// Правило: рейтинг должен быть больше или равен 0, но меньше или равен 10
	Rating float64
}

func (u *User) ChangeName(newName string) {
	if newName != "" {
	u.Name = newName
	}
} // смена имени

func (u *User) ChangeAge(newAge int) {
	if 0 < newAge && newAge < 150 {
	u.Age = newAge
	}
} // смена возраста

func (u *User) ChangePhoneNumber(newPhoneNumber string) {
	if newPhoneNumber != "" {
	u.PhoneNumber = newPhoneNumber
	}
} // смена номера телефона

func (u *User) CloseAccount() {
	u.IsClose = true
} // закрытие аккаунта

func (u *User) OpenAccount() {
	u.IsClose = false
} //открытие аккаунта

func (u *User) RatingUP(rating float64) {
	if u.Rating+rating <= 10.0 {
	u.Rating += rating
	}
} // поднятие рейтинга

func (u *User) RatingDOWN(rating float64) {
	if u.Rating+rating >= 0 {
	u.Rating -= rating
	}
} // опускание рейтинга


func main() {
	user := User{
		Name: "artzilaw",
		Age: 20,
		PhoneNumber: "89882281488",
		IsClose: true,
		Rating: 8.0,
	}
	fmt.Println("Рейтинг до:", user.Rating)
	user.RatingUP(1.2)
	fmt.Println("Рейтинг после :", user.Rating)
}
```

То что выше - это скорее как пример, причем очень хорошо расписанный пример реализации, но проблема в том,что при задании невалидных значений в структуре напрямую или при создании(второе это про в наглую задать условно Rating : 100 для user, а первое это про обратившись через user.Rating и задать ему условно 228, ну или любой другой невалид), про первое пока не могу сказать, но второе можно пофиксить через конструктор(только давайте условимся в том,что пока мы не умеем обрабатывать ошибки, так что если значение невалидно, то возвращаем пустой конструктор)

Реализация:
```go
func NewUser(
	name string,
	age int,
	phoneNumber string,
	isClose bool,
	rating float64,
) User {
	if name == "" {
		return User{}
	}
	if age <= 0 || age >= 150 {
		return User{}
	}
	if phoneNumber == "" {
		return User{}
	}
	if rating < 0 && rating > 10 {
		return User{}
	}
	return User{
		Name: name,
		Age: age,
		PhoneNumber: phoneNumber,
		IsClose: isClose,
		Rating: rating,
	}
}
```

Тоесть, в качестве аргументов мы подали те,которые идентичны по типу нашим полям структуры, далее если хоть где-то невалидное значение - то возвращает пустого юзера, если все хорошо, то полям присваиваются значения конструктора 
Как через него создавать пользователя:
```go
user := NewUser(
	"Artzilaw",
	20,
	"89882281488",
	true,
	2.0,
)
```
Если бы хоть одно значение было невалидно,то опять же вернулся пустой конструктор(ну как пустой, он то не пустой, просто забит значениями по умолчанию)
