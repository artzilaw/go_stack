Указатель -  это переменная, с помощью которой можно будет обращаться к другим объектам в нашей памяти 
Их придумали для того,чтобы оперативу не засорять
Например: У нас есть какая-то многовесная переменная, и при вызове функции и передачи ее,как аргумента, эта переменная копируется, и в итоге у нас съедается очень много памяти
Указатель же очень легковесный, он также выделяется в памяти,но занимает куда меньше места,  и когда нам нужна будет эта переменная, мы к ней можем обратиться с помощью указателя
Но иногда нет возможности обратиться к переменной через указатель,там только копировать и получается,но если надо работать с оригиналом,то без указателя никак,например,если брать конспект по [[05_defer()|defer()]] , то я там приводили пример с подключением к бд, так вот, подключение к бд во всей программе должно быть только одно, во избежании ошибок,поэтому копировать его не получится, вот тут то и помогают указатели 

**==Примечание:==** при изменении исходного объекта через указатель, другие указатели это тоже видят

## Cинтаксис 
```go 
package main

import "fmt"

func main() {
	number := 10
	pointer := &number
	fmt.Println(pointer) // выведет адрес переменной number в памяти
}
```

То есть,исходя из примера выше, указатель(pointer) содержит адрес в памяти, где хранится a(адрес берется через &)

==**Примечание:**== Так как го статически типизированный язык, то и указатель на условно тип int или string это разные указатели

```go
package main

import "fmt"

func main() {
	number := 10
	pointer := &number
	fmt.Println(pointer)
	fmt.Println(*pointer)
}

func foo(n *int){
	
}
```

Что означает * в функции и в чем ее отличие от & в коде выше? 
& -  означает,что мы берем адрес переменной из оперативной памяти, тоесть когда мы пишем *pointer := &number* , мы *pointer* присваиваем адрес переменной * -  в примере выше означает указатель на какой-то тип, тоесть в функции у нас хранится тип данных *указатель на int*

Память выделяется по-разному при разных запусках программы,поэтому при одном запуске выделяется один адрес, при другом другой,поэтому при запуске проги ниже у нас выведется другой адрес(для наглядности прикрепил новый вывод программы со старым)
```go
package main

import "fmt"

func main() {
	number := 10
	pointer := &number
	foo(pointer)
}
func foo(n *int) {
	fmt.Println(n)
}
```

**Разыменовывание  указателя**

Допустим нам надо добраться до переменной,чтобы с  ней выполнить какие-то операции ,для этого надо воспользоваться операцией разыменовывания

```go
package main

import "fmt"

func main() {
	number := 10
	pointer := &number
	foo(pointer)
}

func foo(n *int) {
	fmt.Println(n)
	fmt.Println(*n) // вот она как раз
}
```

## Работа с "оригиналом" переменной в функции

Как было сказано в [[04_Функции | этом]] конспекте, мы в функции можем работать только с копией переменной, но от этого оригинальное значение функции не поменяется, но через указатели можно через функции воздействовать на оригинал
```go
package main

import "fmt"

func main() {
	number := 10
	pointer := &number
	foo(pointer)
	fmt.Println(number)
}

func foo(n *int) {
	fmt.Println(n)
	fmt.Println(*n)
	*n = 5
}

Вывод:
0x23a3615520e0
10 // вывод с foo()
5 // вывод c main()
```

Мы получается в main() объявили указатель на *number := 10*, дальше мы перешли в функцию foo(), там после выводов мы разыменовали указатель( * n  = 5),то есть от адреса перешли к значению  и поменяли это значение, соответственно из-за этого в выводе number в main() у нас уже не 10, а 5.

Но если капнуть чуть глубже,то получается,что мы указатель просто скопировали,но скопированный указатель  указывает на ту же область памяти,что и "оригинал", от чего у нас и получилось разыменовать number

И даже если разыменовать pointer(вывести * pointer после number ), то выведется 5, так как мы опять же через скопированный указатель поменяли значение этой переменной

## Потеря указателя
Так как мы в функции только копируем значения, то следовательно мы в функциях работаем только с копиями, так вот , при передаче в функцию аргумента типа указатель на тот или иной тип, у нас выходит, что копируется указатель, и если аргументу функции в этой же функции передать указатель на что-то другое, то он перестанет указывать на ту же переменную,на которую изначально указывал и будет указывать на то,что мы ему сказали указывать

```go
package main

import "fmt"

func main() {
	name := "Иван"
	ptr := &name
	changeName(ptr)
}

func changeName(s *string) {
	str := "hello"
	s = &str // теперь указывает не на name в main(), а на str в changeName()
}
```

## nil указатели
Представляет из себя указатель,который указывает в никуда,говоря по-другому - значение указателя по-умолчанию

```go
package main

import "fmt"

func main() {
	number := 15
	var ptr *int // указывает в никуда
	fmt.Println(number)
	fmt.Println(ptr)
}

Вывод:
15
<nil> // вывод ptr
```

При попытке разыменовать такой указатель выскочит ошибка,соответсвенно,чтобы ее не было, проводят проверку на nil указатель
 ```go
 if ptr != nil {
	 fmt.Println("Разыменовываем: ", *ptr)
 }else{
 fmt.Println("Получен nil-указатель")
 }
 ```
