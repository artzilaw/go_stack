## Массивы
Массив - это последовательность значений, которая находится в памяти непрерывно
![[massive.png|601]]Что значит непрерывно? Это значит,что если наш элемент расположился на какой-то ячейке памяти, то следующий элемент располагается прямо за ним, как в примере выше

В языке Go это выглядит так:
```go
package main

func main(){
	arr := [5]int{5,66,7,100,1}
	fmt.Println(arr)
}

Вывод:
[5 66 7 100 1]
```

В первых квадратных скобках указываем количество элементов, потом тип  массива , потом сами элементы, причем элементы должны быть того же типа, что и сам массив 

## Нумерация элементов в массиве

Нумерация здесь, как и в других ЯП, идет с 0 до n-1, где n - это количество элементов в массиве

Если брать код выше, то идет следующее соответствие по индексам(индекс - это номер массива по его нумерации):
 0 - 5
 1 - 66
 2 - 7
 3 - 100
 4 - 1
 Почему так? Исторически сложилось, исходя из инфы,что надобыл, там приколы в процессоре со смещением относительно базы, типо пока в базе 0, потом как сделали шаг в 1 и тд, советую самим почитать, если интересно, но нам это сейчас не так сильно надо

## Индексы и работа с диапазоном статического массива
Это все к тому,что мы можем обращаться к элементам массива через индекс
```go
fmt.Println(arr[1]) // fmt.Println(arr[1])выведет 66
```

Также можно совершать арифметические операции над этими самыми элементами:
```go
arr[1] += 5 // прибавляем 5 к 66 
fmt.Println(arr[1]) // 71
```

Если попытаться вывести несуществующий элемент,  тоесть по индексу, которого нет, то выскочит ошибка

```go
fmt.Println(arr[5]) 

Вывод:
# command-line-arguments
./main.go:10:18: invalid argument: index 5 out of bounds [0:5]
```
Консоль нам буквально говорит, что мы пытаемся получить доступ к элементу по индексу, которого не существует
Так как мы уже выделили под массив 5 элементов, за эти 5 элементов(а точнее за память, которую  выделили под массив) мы выходить не можем, а так как элементов 5, то и диапазон индексов `[0,4]` 

Вывод всех 5 **допустимых** значений:
```go
arr := [5]int{5, 66, 7, 100, 1}

fmt.Println(arr[0]) // вывод значения под индексом 0
fmt.Println(arr[1]) // вывод значения под индексом 1
fmt.Println(arr[2]) // вывод значения под индексом 2
fmt.Println(arr[3]) // вывод значения под индексом 3
fmt.Println(arr[4]) // вывод значения под индексом 4

Вывод:
5
66
7
100
1
```


В статическом массиве(с заданным размером) напрямую изменить его размер нельзя, если мы хотим поменять его размер, надо создать новый, перекопировать элементы со старого туда и только потом добавить новый элемент. Это все так муторно,чтобы не занимать ячейку в памяти,которая потенциально отдана под другие задачи, ниже представлены картинка и  реализация описанного
![[massive_static_append.jpg]]
```go
arr2 := [6]int{}
arr2[0] = arr[0]
arr2[1] = arr[1]
arr2[2] = arr[2]
arr2[3] = arr[3]
arr2[4] = arr[4]
arr2[5] = 4
```

## Взаимодействие массива с циклами

Допустим у нас есть задача, по условию которой надо удвоить все значения, которые четные

В теории мы могли бы это сделать через операторы ветвления
```go
if arr[0]%2 == 0 {
	arr[0] *= 2
}

// и так с остальными элементами arr[1],arr[2] и тд
```

Но представим, что массиве не 5 элементов, а 100, 1000, 10000...

Тут уже не сделаешь через ветвления(по крайней мере быстро), так что на помощь приходят циклы:
```go
arr := [5]int{5, 66, 7, 100, 1}
for i := 0; i < 5; i++ {
	if arr[i]%2 == 0 {
		arr[i] *= 2
	}
}
```

Что мы тут сделали? Мы идем от 0, так как нам надо пройти по всему массиву,а нумерация начинается с 0, и получается i = 0, далее мы приходим к блоку ветвления,  там через i=0 идет обращение к первому элементу массива(`arr[i]`), идет сравнение по условию, если true, то производим операцию над элементом массива, с остальными элементами вплоть до i = 4 также

По итогу наши элементы действительно умножаются(кто бы мог подумать):

```bash
[5 68 7 102 1]
```

Выводить кстати тоже можно через цикл,чтобы было лаконичнее
```go
for i := 0; i < 5; i++ {
	fmt.Println(i, "-", arr[i])
}

Вывод:
0 - 5
1 - 68
2 - 7
3 - 102
4 - 1
```

Сейчас приведу более осмысленный пример,который в видосе описан

У нас есть какая-то сущность User{}, где хранится имя, рейтинг и наличие премиума, мы через нее создаем 3 пользователей, нам надо для этих 3 пользователей проверить на наличие премиума, если он есть,то +1 к рейту:
```go
package main

import (
"fmt"
"github.com/k0kubun/pp" //  это просто как дополненение
)

type User struct {
Name string
Rating float64
Premium bool
}

  

func main() {

user1 := User{
	"Даниил",
	6.7,
	true,
}

user2 := User{
	"Шафи",
	1.2,
	false,
}

user3 := User{
	"Мага",
	8.8,
	true,
}

fmt.Println("До")
fmt.Println("-------------")
pp.Println(user1)
pp.Println(user2)
pp.Println(user3)
fmt.Println("")

if user1.Premium {
	user1.Rating++
}

if user2.Premium {
	user2.Rating++
}

if user3.Premium {
	user3.Rating++
}

fmt.Println("После")
fmt.Println("-------------")
pp.Println(user1)
pp.Println(user2)
pp.Println(user3)
fmt.Println("")
}
```  

Но что если количество пользователей увеличится? Не создавать же еще 100 пользователей, поэтому можно закинуть всех этих юзеров в массив
```go
userArray := [3]User{user1,user2,user3}
```

Но проблема в том,что мы будем воздействовать не на сами переменные,а на их копию в памяти, это можно решить в лоб - то есть,просто создавая пользователей в массиве:

```go
userArray := [3]User{

	},

```

Есть наверное и другие способы, но этот надежный, так как у нас всего один источник правды

Теперь через циклы можно как раз массив пользователей обрабатывать просто обратившись через индекс к значению массива(пользователю) и через него к нужному полю, аналогично с алгоритмом увеличения рейтинга:

## Дополнение к для более эффективной работы с массивами 

У нас есть возможность не проверять каждый раз длину массива, это не очень удобно, вместо этого можно просто использовать функцию len(), которая возвращает нам длину массива
```go
for i := 0; i < len(userArray); i++ {
	if userArray[i].Premium {
		userArray[i].Rating++
	}
}
```
Целесообразность ее использования обусловлена тем, что нам не надо в коде менять фиксированное значение, если количество элементов массива уменьшилось/увеличилось, все сводится к тому, что можно банально поменять количество элементов а массиве при его создании(ну и добавлении новых данных)

Также мы можем объявить цикл `for` иначе:
```go
for i, v := range userArray {
	pp.Println(i, v)
}
```

Что означает эта запись? Мы задаем нашему циклу 2 переменные (i и v),где i - индекс массива,  v - значение по индексу; с помощью range мы задаем диапазон, в данном случае в качестве диапазона задан весь массив userArray, ну и в теле работы цикла выводит индекс со значением 

Если нам не надо никаких манипуляций совершать над индексом,а нужно только значение, то мы можем использовать такой вид записи:
```go
for _, v := range userArray {
	pp.Println(v)
}
```
Конструкция выше также работает, если нам нужен индекс,но не значение, тогда просто вместо переменной для значения ставим `__`


Используем теперь эту конструкцию на алгоритме повышения рейтинга  :
```go
for _, user := range userArray {
	if user.Premium {
	user.Rating++
	}
}
```
И посмотрим на результат
![[Pasted image 20260717222403.png]]

Как мы видим, ничего не изменилось, почему так? А все просто, мы в этом цикле работаем с копией, то есть на каждой итерации у нас создается новый user и происходят манипуляции над ним, и живет эта переменная, пока не завершилась работа итерации тела цикла 

Чтобы все работало и изменялось, нужно обращаться через индекс напрямую к элементу массива,то есть делать так:
```go
for index, user := range userArray {
	if user.Premium {
		userArray[index].Rating++
	}
}
```

Что кому использовать решать вам, кто хочет использует старый вариант написания цикла, кто хочет новый, но как по мне старый в плане работы с значениями массива(где не нужен индекс) проигрывает новому, но при работе с новым новичку надо постоянно держать в голове,что он работает с копией(это уже должно прийти с опытом) 

## Слайсы

^2b38bf

Массивы штука конечно удобная, но проблема в том, что он статический, и если надо что-то поменять, то приходится создавать новый, копировать нужные значения со старого и заниматься прочей тягомотиной

Но решение есть,  и название ему **слайсы**

Слайс из себя представляет структуру, которая под капотом условно создает новый массив, перекопирует со старого в новый элементы и добавляет новый, но для нас, как разработчиков, это все выглядит сильно легче(в плане восприятия), выглядит структура как-то так:
![[slice_struct.png]]

Если описывать каждое поле по отдельности, то указатель - это указатель на массив в памяти(ну буквально читается как указатель на массив с тем или иным типом), он нужен для управления реальными данными,а не копиями,  len - количество элементов, а вот насчет поля cap стоит рассказать подробнее.

 У нас под капотом, когда мы добавляем еще один элемент(это делается методом),  у нас массив копируется в другую область памяти, копирует туда все элементы старого массива, но количество элементов массива увеличилось вдвое
 
То есть если у нас изначально было 3 элемента и мы добавили туда 1, то создастся массив с вместимостью 6 элементов, в итоге получается, что len = 4, а  cap = 6(cap это вместимость), и в этот же момент указатель перезаписывапется, он теперь указывает на новый массив, а не старый.И так можно будет добавить в слайс еще 2 элемента, если же больше, то он опять перекопируется, увеличится в 2 раза и там будет len = 7 и cap = 12,указатель будет указывать на новый массив и так далее

Отсюда следует,что если len > cap, то создается массив в 2 раза больше старого(по вместимости) и туда копируются старые элементы + добавляются новые 

## Работа со слайсами в Go

Если мы хотим превратить наш массив из примера [[#^a47227 | выше]] в слайс, достаточно просто убрать статически заданный размер(проще говоря убрать 3) 
```go
userArray := []User{
	User{
		"Даниил",
		6.7,
		true,
	},
	User{
		"Шафи",
		1.2,
		false,
	},
	User{
		"Мага",
		8.8,
		true,
	},
}
```
Что поменяло? А то,что нам теперь доступны функции слайса, теперь мы можем, например, с помощью функции append() добавить новый элемент и выглядит это следующим образом:
```go
userArray = append(
	userArray, 
	User{
		Name: "Васян",
		Rating: 2.0,
		Premium: true,
	},
)

// небольшое дополнение, в примерах кодов выше я не указывал явно какому полю что указывать, а просто указывал сразу значения, так вот, так тоже можно, потому что я указывал их друг за другом, но для наглядности тут поля указая явно
```

То, что было описано [[#^2b38bf | тут ]] , это по сути как у нас работает функция append(), да и тут видно,что мы userArray присваиваем новый слайс, где через append(), который принимает в качестве аргументов  массив и элемент, который надо добавить,  создается новый слайс, состоящий из старых и нового элементов
И как мы видим, у нас теперь 4 элемента в userArray:
![[slice_append_result.png]]
Если вывести длину и вместимость до и после добавления элементов таким образом:
```go
fmt.Println("len:", len(userArray))
fmt.Println("cap:", cap(userArray))
```
 То на выводе видно,что слайс работает ровно также, как я описывал выше
 ``` bash
len: 3
cap: 3
len: 4
cap: 6
 ```

Как еще можно создавать слайсы? Через функцию make
```go
intSlice := make([]int, 0) // если выводить, то в выводе будет []
```
В качестве первого аргумента передается сам массив и его тип, в качестве второго - размер низлежащего массива, если не углубляться, то минимальный cap (так как у нас тут 0, то он создаст после добавления первого элемента массив с объемом 1, потом массив с объемом 2, потом с размером 4 и тд) 
После добавления элементов и вывода должно быть понятнее:
```go
intSlice = append(intSlice, 10, 12, 12, 15)
fmt.Println(len(intSlice), cap(intSlice)) // в выводе будет 4 и 4
```

Первое что видно,в append() можно передавать сразу несколько новых элементов, а второе,что длина и вместимость работают так, как должны

Если указать в качестве второго аргумента какой-то другое число,допустим 5, то у нас нас будет размер низлежащего массива 5, как элементов станет больше 5, то увеличится до 10 и так далее 
```go
intSlice := make([]int, 5)
fmt.Println(intSlice) // [0 0 0 0 0]
```

Но такой кейс - это редкость, и если нам надо выделить какое-то число элементов, то делают так:
```go
intSlice := make([]int, 0, 5)
``` 

Что поменялось? А то, что у нас теперь массив заполняется не нулями, а ничем, то есть len = 0, cap = 5(в прошлом примере и то , и то было равно 5) и небходимости там очищать массив от 0, перевыделять память при увеличении количества элементов и заход за len не будет 
![[slice_empty_make.png]]

## Мапы

Мапы представляют из себя структуры данных,которые получают доступ к значению не по индексу, а по какому-то ключу, вот на примере ниже у нас условный прогноз погоды, мол в какое-то число будет столько то градусов, в качестве ключа выступает число, а в качестве значения градус
![[maps_example.png|155]]

В теории это можно было бы все реализовать или через переменные, или через массив,но если делать через переменные,то там будет просто каша, учитывая что это все еще масштабировать надо, а если через массив, то удобнее, но не то что нам нужно(держать в голове, что в первом индексе у нас погода на условное 11 число)
```go
Реализация через переменные:
weather11 := +3
weather12 := +6
weather13 := +9
weather14 := -4
weather15 := +1

fmt.Println(weather11)
fmt.Println(weather12)
fmt.Println(weather13)
fmt.Println(weather14)
fmt.Println(weather15) // тут и так все понятно

Реализация через массив:

weather := []int{
	+3,
	+6,
	+9,
	-4,
	+1,
}

for _, v := range weather {
	fmt.Println(v)
} // как тут понять, где какое число
```

Поэтому у нас существует такая структура данных, как мапа, в которой, если дальше рассматривать наш пример, можно задать число, как ключ, градус, как значение.
Она использует формат данных  key -> value, тоесть, ключ значение, если говорить кратко, то он утверждает, что у нас есть ключи, по эти ключам можно получить значение
Чтобы ее объявить, надо написать ключевое слово map, потом в квадратных скобках указать тип ключа, а после них тип значения, в примере ниже они оба int:
```go
weather := map[int]int{
	11: +3,
	12: +6,
	13: +9,
	14: -4,
	15: +1,
}

fmt.Println(weather[12]) // выведет 6, так как значение у нас 6
```

Если вывести ключ, который не задан, то выведется значение по-умолчанию типа значение, тоесть если тип значения int,то выведется 0, если string, то "" и тд
```go
fmt.Println(weather[30]) // выведется 0
```
Но если мы добавим 30 в нашу мапу, и там будет значением 30, то как его отличить от значения по-умолчанию?
```go
weather := map[int]int{
	11: +3,
	12: +6,
	13: +9,
	14: -4,
	15: +1,
	30: 0,
}
```
Для этого есть такое значение в Go, как ok, это значение типа bool, и если значение у нас валидное(не значение по умолчанию), он должен вывести true
```go
c, ok := weather[30] // закинули значени по ключу 30 в c и через ok проверяем, валидное ли оно
fmt.Println(c, ok) // выведется 0 и true
```
Так как это переменная типа bool, то можно завязаться на ветвлениях
```go
if ok{
	fmt.Println("Okay") // в нашем случае выведется это
}else {
	fmt.Println("bad")
}
```

Добавлять элементы в мапу довольно просто, нам надо всего лишь присвоить новому ключу значение, для наглядности пример ниже:
```go
weather := map[int]int{
	11: +3,
	12: +6,
	13: +9,
	14: -4,
	15: +1,
	30: 0,
}
weather[20] = -10 // вот наше добавление
fmt.Println(weather[20])
```

Если сделать так с существующим ключом, то существующее значение просто перезапишется:
```go
weather := map[int]int{
	11: +3,
	12: +6,
	13: +9,
	14: -4,
	15: +1,
	30: 0,
}

fmt.Println(weather[11]) // тут будет 3
weather[11] = -10 
fmt.Println(weather[11]) // тут будет -10
```

## Работа с циклами в мапе
Допустим к нам пришла информация о том, что с 11 по 15 число температура повысится на 1 градус, чтобы это реализовать можно использовать циклы, причем мы можем использовать только циклы нового формата(старого увы нельзя):
```go
weather := map[int]int{
	11: +3,
	12: +6,
	13: +9,
	14: -4,
	15: +1,
}

for k, v := range weather {
	fmt.Println(k, v)
}
```

Но вывод у нас будет какой-то странный:
```bash
15 1
11 3
12 6
13 9
14 -4
```

У нас сначала последний элемент выводится, потом остальные, почему так?
Это связано с тем, что по внутреннему устройству мапы, нет гарантии того, что порядок будет верен, это не массив, нам в принципе должно быть все равно на порядок,нам главное,чтобы сохранилась привязка ключа к значению

Если вернуться к нашей задаче, то мы можем сделать так:
```go
for k, _ := range weather {
	weather[k] += 1
}
for _, v := range weather {
	fmt.Println(v) 
}
```

Почему мы не сделали через v+=1? а все просто, у нас бы просто началась работа с копиями и ничего бы не изменилось, все как в слайсах

Еще мапы можно создавать так
```go
weather := map[int]int{}
weather[7] = 12
weather[10] = 15
```

То есть, не заполняя значениями изначально, а добавляя потом, если надо

Также можно через make:
```go
weather := make(map[int]int) //без задания размера
weather := make(map[int]int, 10) // с заданием размера
```

Принцип работы такой же, как и в мапе, без выделения у нас будет автоматически выделяться память, с выделением какое-то количество памяти нам уже выделено(но через cap() получить значение вместимости увы нельзя)

Для более наглядно примера работы с ними возьмем задачку, по которой у нас есть база каких-то людей с судимостью и по ней надо понять, судим он или нет, тут в качестве ключ-значения возьмем имя и булеву переменную, которая показывает, был судим ли человек
Далее мы присваиваем c и ok значение ключа "Васян"(можно взять и другой), ok у нас проверяет, есть ли вообще по такому ключу какое-либо значение, если нет, то завершает программу по return, если есть, то смотрим через c судим ли он и отталкиваясь от вывода выводим соответствующий текст"
```go
package main

import "fmt"

func main() {

criminal := map[string]bool{
	"Васян": true,
	"Витек": false,
	"Шафи": true,
	"Мага": false,
	"Абукурах": true,
}

c, ok := criminal["Васян"]
if !ok {
	fmt.Println("Человека нет в базе")
	return
}

fmt.Println("Человека найден в базе")

if c {
		fmt.Println("Человек судим")
	} else {
		fmt.Println("Человек не судим")
	}
}
```
