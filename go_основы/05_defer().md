Для начала надо проговорить, что такое Стек

Стек , проще говоря, то какой-то набор данных,куда можно класть данные или забирать оттуда данные, эти данные и кладутся сверху и забираются сверху(мог бы визулизировать,но почему-то у меня картинки в обсидиане не вставляются), тоесть,что зашло первым,уйдет последним	![[stack.png]]
defer() же из себя представляет анонимную функцию и работает по принципу того,что выполняется последним в объявленной функции

```go
package main

import "fmt"

func main() {
	fmt.Println("Начало функции main")
	defer func() {
		fmt.Println("Конец функции main")
	}()
	hello()
}

  

func hello() {
	fmt.Println("Начало функции hello")
	defer func() {
		fmt.Println("Конец функции hello")
	}()
	fmt.Println("hello1")
	fmt.Println("hello2")
	fmt.Println("hello3")
}

Вывод:
Начало функции main
Начало функции hello
hello1
hello2
hello3
Конец функции hello
Конец функции main
```
Как видно из вывода, сначала у нас срабатывает начало main(), потом, идя по  main(), вызывается hello(), в функции hello() начинается выполнение,функция заканчивает работать и вызывается defer(), который **внутри функции hello()**, после этого,мы дальше идем по main и у нас уже вызывается defer()? который в  main()

## Несколько defer() в одной функции

Мы можем вызывать несколько defer()

```go
package main

import "fmt"

func main() {
	fmt.Println("Начало функции main")
	defer func() {
		fmt.Println("Конец функции main")
	}()
	foo()
}
func foo() {
	defer func() {
		fmt.Println("Defer1")
	}()
	defer func() {
		fmt.Println("Defer2")
	}()
	defer func() {
		fmt.Println("Defer3")
	}()
	fmt.Println("foo")
}

Вывод:
Начало функции main
foo
Defer3
Defer2
Defer1
Конец функции main
```

Как видно из вывода, выполняется изначально последний из объявленных defer(), а первый объявленный выполняется последним, так как эта самая функция кладется в стек отложенных вызовов
