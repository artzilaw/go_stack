Цикл - многократное повторение какого-то алгоритма  
Пример цикла в языке go :
```go
package main

import "fmt"

func main() {
	for i := 1; i <= 5; i++ {
		fmt.Println("Hello World!")
	}
}

//Вывод:
//Hello World!
//Hello World!
//Hello World!
//Hello World!
//Hello World! 
```

Как мы видим, у нас повторяются выводы 5 раз, но почему именно так?
Сначала мы должны объявить переменную, за это отвечает участое  *i := "значание"*  , далее у нас идет условие *i <= "значение"*(может быть любой знак),то есть,пока не у нас true, тело цикла работает,как только у нас false, цикл прерывается и программа выполняется дальше, далее идет шаг,в примере выше указан как инкремент, он и увеличивает значение i, прога выше по ходу выполнения программы имеет следующие значения:
i = 1 <= 5 ; i++ - идем в цикл
i = 2 <= 5 ;  i++ - идем в цикл
i = 3 <= 5 ;  i++ - идем в цикл
i = 4 <= 5 ;  i++ - идем в цикл 
i = 5 <= 5 ;  i++ - идем в цикл
i = 6 <= 5 ;  i++ - стоп,так как 6 не меньше 5

и того,цикл у нас отработал 5 раз, интересно,что при секциях i := 0; i < 5; i++ тоже было бы 5 раз,i:=0 также считается за итерацию 

Ксатати, можно в принципе с областью видимости экспериментировать ,  возьмем как условие цикла i := 1; i <= 10; i += 2:
```go
for i := 1; i <= 10; i += 2 {
	fmt.Println("Hello world", i)
}

Вывод:
Hello world 1
Hello world 3
Hello world 5
Hello world 7
Hello world 9
```
Почему так? У нас i с шагом 2, тоесть значение повышается на 2:
*i = 1+2*
*i = 3 +2*
*и тд*
И так доходит пока условие i<=10 не будет false
## Область видимости 
Если мы будем выводить i, то у нас будет выводиться значение,в зависимости от итерации:
```go
package main

import "fmt"

func main() {
	for i := 1; i <= 5; i++ {
		fmt.Println("Hello World!", i)
	}
}
// Вывод:
//Hello World! 1
//Hello World! 2
//Hello World! 3
//Hello World! 4
//Hello World! 5
```
 Как мы видим, выводится корректно,но стоит вывести i за пределами тела цикла,то выскочит ошибка:

```go 
for i := 1; i <= 5; i++ {
	fmt.Println("Hello World!", i)
}

i = 5
fmt.Println(i)

//Вывод:
//./main.go:12:30: undefined: i
```
Почему так?  Дело в том,что i существует только для тела цикла,то есть не смотря на то,что мы ее создали,но создали в теле цикла, за его пределами i не существует

Более того,можно создать переменные внутри тела цикла и они будут рабочими,но опять же, за его пределами такой переменной не существует:
```go
package main

import "fmt"

func main() {
	for i := 1; i <= 5; i++ {
		score := 5
		fmt.Println("Hello World!", i, score)
		score += 1  
	}
}

Вывод:
Hello World! 1 5
Hello World! 2 5
Hello World! 3 5
Hello World! 4 5
Hello World! 5 5
``` 
 Но почему переменная score не меняется? А все просто,переменная существует только одну итерацию, после окончания итерации, переменная удаляется,от чего score постоянно равно 5(если до вывода допустим прибавить 3,то при выводе будет выводиться 8):
 ```go
 package main

import "fmt"

func main() {
	for i := 1; i <= 5; i++ {
		score := 5
		score += 3
		fmt.Println("Hello World!", i, score)
		score += 1
	}
}
Вывод:
Hello World! 1 8
Hello World! 2 8
Hello World! 3 8
Hello World! 4 8
Hello World! 5 8
 ```
 Если возьмем условно переменную number, которую объявим в main(), то если ее как-либо менять в цикле,то она будет меняться , тоесть, для переменной number видимость это вся функция main() :
 ```go
 package main

import "fmt"

func main() {
	number := 0
	for i := 1; i <= 5; i++ {
		score := 5
		score += 3
		fmt.Println("Hello World!", i, score)
		score += 1
		fmt.Println(number)
		number++  
	}  
}
Вывод:
Hello World! 1 8
0
Hello World! 2 8
1
Hello World! 3 8
2
Hello World! 4 8
3
Hello World! 5 8
4
 ```

## Бесконечный цикл
 Представляет из себя цикл с бесконечным количеством итераций, по сути так как условия нет, из него можно выбраться только через break(об этим ниже)
 
 Вид:
 ```go
 for{
	 // Ваш код
 }
 ```

## break
Прерывает цикл,будь то бесконечный или нет цикл

Вид(пусть будет с бесконечным циклом и каким-то условием:
```go
for{
	// Код
	if условие {
		break
	}
	
}
```
