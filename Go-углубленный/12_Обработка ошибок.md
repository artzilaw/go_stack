В качестве примера возьмем прогу, где есть  сущность пользователя, у этого пользователя есть имя и баланс, также у нас есть функция(именно функция, не метод, хз почему не метод), которая принимает указатель на экземпляр структуры и какой-то целочисленный баланс:
```go
package main

import "github.com/k0kubun/pp"

type User struct {
	Name string
	Ballance int
}

  

func NewUser(name string, ballance int) User {
	return User{
		Name: name,
		Ballance: ballance,
	}
}

func Pay(user *User, usd int) {
	user.Ballance -= usd
}

  

func main() {
	user := NewUser("Олег", 10)
	Pay(&user, 5) // тут принимаем по функции укзатель
	pp.Println(user)
}
``` 

Если вывести, то все будет работать
![[error_result.png]]
Но что если в качестве второго аргумента в функцию передать 50 `Pay(&user, 5)`, тогда у нас будет поле Ballance : -40, такого доcпукать нельзя

По идее можно сделать так:
```go
func Pay(user *User, usd int) {
```

Тогда функция не заработает, деньги не спишутся, но с точки зрения пользователя это будет выглядеть, как то, что он попытался снять деньги, а не изменилось ничего и непонятно, была там покупка, не было ее

## А вот и сам error
Для индикации ошибки придумали интерфейс error ; из себя он представляет метод Error, возвращающий строку
![[error_structure.png]]

Теперь явно понятно,что у нас возвращаемое значение не что попало, а именно ошибка:
```go
func Pay(user *User, usd int) error {
	if user.Ballance < 0 {
		return errors.New("Недостаточно средств")
	}
	user.Ballance -= usd
	return nil
}

func main() {
	user := NewUser("Олег", 10)
	pp.Println(user)
	err := Pay(&user, 50)
	pp.Println(user)
	if err != nil {
		fmt.Println("Не Оплачено", err.Error())
	} else {
		fmt.Println("Оплачено")
	}
}
```

Что произошло в коде выше? Мы задали функции значение интерфейса error, потом после проверки мы вернули `errors.New("Недостаточно средств")`, как я понял, эта функция просто принимает текст и  возвращает указатель на  структуру errorString,  при успешном выполнении программы мы возвращаем nil(то есть, говорим что все ок), в самом коде мы присваиваем переменной err возвращенное значение функции и если там не nil, то выводится ошибка через метод Error(), если все хорошо, то выводится наш текст

Тоесть, просто сигнализирует, была ли ошибка, если нет, то возвращается nil

## Почему мы можем сравнивать интерфейс с указателем?
Для начала надо бы разобраться что из себя представляет nil, следуя из описания - это идентификатор, который представляет из себя нулевое значение для интерфейса, указателя,функции, мапы и тд
![[nil_explanation.png]]

 То есть, если у какого-то интерфейса тип nil, то под этим интерфейсом не лежит никакого объекта 

Сам же интерфейс имеет под собой 2 указателя, один на тип объекта, другой на сам объект
![[interface_pointer.png]]

Если под интерфейсом возвращается nil, то и тип объекта nil и сам объект nil, так и получается нулевой интерфейс, тоесть, если у нас не nil, то это значит что в интерфейсе что-то есть и в контексте ошибки - это текст ошибки 

Также бывают ситуации, когда тип объекта определяется, а сам объект не определяется(но об этом не сейчас, потом где-нибудь почитаю) 

## Возвращение помимо ошибки еще какого-то значения

У нас есть структура под названием Car, у нее есть поле брони Armour int и метод Gas()(int,error), как видно, тут нужно возвращать 2 значения, сама функция предмтавляет из себя генерацию рандомного числа, которое обозначает значение нашего разгона, но при каждом таком рывке уменьшается прочность на 10
Но тут возникает проблема, прочность не может быть отрицательной, и чтобы на таковой не стала, ее надо обработать, то есть вернуть 2 значения, причем нам больше интересно второе: при ошибке, в него вернется ошибка `errors.New("прочности не осталось")`, то есть в интерфейс запишется тип объекта(строка) и сам объект(текст ошибки), если все хорошо, то вернется nil, в go это выглядит так:
```go
func (c *Car) Gas() (int, error) {
	if c.Armor-10 <= 0 {
		return 0, errors.New("прочности не осталось")
	}
	launch := rand.Intn(150)
	c.Armor -= 10
	return launch, nil
}

func main() {
	car := NewCar(25)
	pp.Println("Машина до:", car)
	kmh, err := car.Gas()
	if err != nil {
		fmt.Println(err.Error())
	} else {
		fmt.Println("Получившийся разгон:", kmh)
	}
	pp.Println("Машина после:", car)
}
```
